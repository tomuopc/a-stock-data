# 转实盘策略

将回测策略文件转换为实盘可执行版本。用户需提供：原 py 文件、新 py 文件（输出目标）、新 csv 文件（交易记录目标）。

## 流程

1. **复制原文件**：将原 py 文件原封不动复制到新 py 文件
2. **修改新 py 文件**：
   - 去除所有图表输出（`plt.show`、`plt.savefig`、`matplotlib` 相关绘图代码）
   - 去除所有文字回测报告输出（`print` 中的绩效统计、收益率、回撤等）
   - 去除回测循环、净值计算、资金管理、持仓市值计算等回测逻辑
   - **保留**：数据获取、信号计算、选股逻辑、指标计算、买卖判断等核心逻辑
3. **输出今日操作指令**：在文件末尾替换为以下输出格式：

```
今日（<当前日期>）操作：
标的A：买入（原因：<原因>）
标的B：卖出（原因：<原因>）
标的C：持有（原因：<原因>）
```

4. **交易记录到 CSV**：如果今日发生了买入或卖出操作，自动追加到指定的 csv 文件中。CSV 列结构如下：

| 列名 | 说明 |
|------|------|
| time | 交易日期 `YYYY-MM-DD` |
| symbol | 标的代码/名称 |
| action | 操作类型：`买入` / `卖出` |
| volume | 数量 |
| price | 成交价格 |
| reason | 交易原因 |
| hold_days | 持有天数（卖出时填写，买入时为 0） |
| pnl | 盈亏金额（卖出时填写，买入时为 0） |

## 输出示例

```
今日（2026-06-30）操作：
贵州茅台：买入（原因：突破20日均线，成交量放大）
五粮液：持有（原因：仍在上涨趋势中）
宁德时代：卖出（原因：跌破支撑位，止盈出场）

交易记录已保存到：交易记录_实盘.csv
```

## 执行规则

1. **先复制，后修改**：原文件必须原封不动保留，所有修改只在新文件上进行
2. **只删不改逻辑**：删除回测相关代码时，不要修改核心信号/选股/指标的逻辑
3. **CSV 追加**：如 csv 文件已存在且有表头，追加行数据；如文件不存在或为空，新建文件并写入表头
4. **保留 import**：不要删除任何 import 语句，即使某些包只在回测中使用

# 策略代码审查

对指定的 Python 回测策略文件进行全面的代码审查，检测回测中常见的逻辑错误和潜在问题。

## 使用方式

用户指定一个 `.py` 文件路径，AI 将对该文件进行全面扫描，并输出一份详细的分析报告。

**重要：AI 不得对代码进行任何修改，仅输出分析报告。**

## 审查项清单

### 1. 未来函数检测
- 检查是否使用了未来数据（如当日的收盘价用于当日开盘买入决策）
- 检查信号计算中是否使用了不应在当前 K 线可获取的数据
- 检查 `shift`、`iloc`、`rolling` 等操作是否存在前瞻偏差
- 检查指标计算中是否混入了未来数据

### 2. 无限流动性检查
- 检查是否假设可以按任意价格成交任意数量
- 检查是否考虑了成交量/持仓量的限制
- 检查是否对涨跌停板上的交易进行了处理
- 检查是否存在仓位管理或资金管理约束

### 3. 涨跌停处理检查
- 检查是否考虑了 A 股 ±10% 涨跌停限制（科创板 ±20%，北交所 ±30%）
- 检查在涨停时是否仍假设可以买入
- 检查在跌停时是否仍假设可以卖出

### 4. 过拟合风险检查
- 检查参数数量与数据量的比例是否合理
- 检查是否存在大量硬编码参数
- 检查是否有多参数优化但未做样本外验证
- 检查策略逻辑是否过于复杂而数据量不足

### 5. 交易成本检查
- 检查是否考虑了佣金、印花税、滑点
- 检查交易成本设置是否合理

### 6. 其他常见问题
- 检查索引对齐是否可能导致数据偏移
- 检查除零、NaN 等数据异常处理
- 检查循环效率及大数据量下的性能问题
- 检查信号与交易执行之间是否存在逻辑缺口

## 报告格式

审查完成后输出以下格式的报告：

```

## 文件：<文件路径>

### 1. 未来函数检测
- 风险等级：[无/低/中/高]
- 发现的问题：[详细描述，无则写"未发现明显问题"]

### 2. 无限流动性检查
- 风险等级：[无/低/中/高]
- 发现的问题：[详细描述]

### 3. 涨跌停处理检查
- 风险等级：[无/低/中/高]
- 发现的问题：[详细描述]

### 4. 过拟合风险检查
- 风险等级：[无/低/中/高]
- 发现的问题：[详细描述]

### 5. 交易成本检查
- 风险等级：[无/低/中/高]
- 发现的问题：[详细描述]

### 6. 其他问题
- 风险等级：[无/低/中/高]
- 发现的问题：[详细描述]

## 综合评估
![高风险|中风险|低风险]

## 修改建议
- [具体建议列表，如无可写"无需修改"]
```

## 执行规则

1. **只读不写**：不得对代码进行任何增、删、改操作
2. **完整报告**：覆盖以上全部审查项，每个审查项必须给出明确的风险等级
3. **实事求是**：没有发现问题就写"未发现明显问题"，不要强行编造问题
4. **有理有据**：每个问题必须引用具体的代码行号作为依据

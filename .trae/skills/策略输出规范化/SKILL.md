# 策略输出规范化

对指定的 Python 回测策略文件进行分析，在其回测输出末尾追加统一的绩效报告格式，不影响原有的任何逻辑和输出。

## 使用方式

用户指定一个 `.py` 文件路径，AI 将：
1. 读取并理解该文件的回测逻辑和已有的输出内容
2. **保持原代码逻辑完全不变**，不在任何中间位置插入代码
3. 仅在文件末尾（所有回测计算完成后、主流程结束前）追加以下标准化绩效输出：

```
--- 回测绩效报告 ---
  "期初净值": ,
  "期末净值": ,
  "平均年化": ,
  "最大回撤": ,
  "最大回撤开始时间": "xxxx-xx-xx",
  "最大回撤结束时间": "xxxx-xx-xx",
  "最大未创新高天数": ,
  "每笔平均盈亏比":
  "单笔最大盈利比": ,
  "单笔最大亏损比": ,
  "最大盈利次数": ,
  "最大亏损次数": ,
  "总交易次数": ,
  "胜率": ,
  "夏普":
```

## 输出字段说明

| 字段 | 说明 |
|------|------|
| 期初净值 | 回测起始时的账户净值 |
| 期末净值 | 回测结束时的账户净值 |
| 平均年化 | 年化收益率（百分比形式） |
| 最大回撤 | 最大回撤幅度（百分比形式） |
| 最大回撤开始时间 | 最大回撤区间的起始日期 |
| 最大回撤结束时间 | 最大回撤区间的结束日期 |
| 最大未创新高天数 | 从净值高点回落未创新高的最长天数 |
| 每笔平均盈亏比 | 所有已平仓交易的盈亏比平均值 |
| 单笔最大盈利比 | 单笔交易的最大盈利比例 |
| 单笔最大亏损比 | 单笔交易的最大亏损比例 |
| 最大盈利次数 | 盈利交易的总次数 |
| 最大亏损次数 | 亏损交易的总次数 |
| 总交易次数 | 所有交易的总次数 |
| 胜率 | 盈利交易次数占总交易次数的比例（百分比形式） |
| 夏普 | 夏普比率 |

## 执行规则

1. **零侵入**：不修改、不删除、不重构原有代码的任何逻辑
2. **仅追加**：只在文件末尾合适位置插入绩效报告的输出代码
3. **数据推导**：根据策略已有的净值序列或交易记录来计算以上指标；如果某些指标在代码中已有计算结果，优先复用
4. **输出位置**：插入在回测主流程结束之前，确保所有回测计算已完成
